# 00 Overview

## 系統目標
- 建置「倉儲管理系統（碼頭系統）」作為既有配送管理系統的擴充模組，支援倉儲人員日常出貨/收貨/退貨作業，並與既有後台及運務 App 協作即時同步。
- 以「一眼即辨」的狀態視覺化與即時同步（狀態更新 < 3 秒）取代既有 Excel/電話追蹤流程。

## 使用者角色
- 倉儲人員：前台主要使用者（登入後進行公告、出貨、收貨、退貨作業）。
- 後台管理員（總後台）：管理碼頭與人員、查詢收貨資訊、監控資料與同步狀態。
- 運務人員（司機，透過運務 App）：接收推播通知、回報「回廠下貨」、進行簽名確認。

## MVP 功能模組（依 PRD/Epics）
- 人員管理：登入/登出、忘記密碼提示資訊顯示、依內部廠區資料隔離。
- 即時公告：公告列表/詳細、已讀/未讀、刪除、依篩選條件的批量已讀/刪除。
- 出貨作業管理：碼頭儀錶板、未排車次列表、狀態變更（完成出車/取消/缺貨待料/通知未到）、通知運務。
- 收貨作業管理：收貨列表 → 配送點總表 → 收貨明細（三層架構）、下貨狀態追蹤、下貨碼頭選擇、簽名送出後完成鎖定。
- 退貨作業管理：退貨單列表/詳細、依狀態切換編輯/查看、來源整合（運務 App 與後台）。
- 即時同步與通知：前後台狀態即時同步、多裝置同步、推播通知、斷線提示與恢復。
- 後台管理（部分）：碼頭管理（碼頭列表/碼頭人員列表/退貨單列表）、倉儲收貨資訊查詢、前後台資料同步查看與依權限修改。
- 資料整合與追溯：與既有後台資料格式一致、狀態變更操作記錄、可追溯性與資料完整性驗證。

## 功能模組對應 Epic
- Epic 1：後台基礎資料管理（碼頭與人員）
- Epic 2：身份驗證與基礎設定（登入/登出/忘記密碼提示/廠區資料隔離）
- Epic 3：即時公告管理
- Epic 4：出貨作業管理
- Epic 5：收貨作業管理
- Epic 6：退貨作業管理
- Epic 7：即時同步與通知基礎設施
- Epic 8：後台管理功能擴充
- Epic 9：資料整合與審計

## 範圍邊界
- In Scope（MVP）：上述 MVP 功能模組與其必要的整合/同步行為。
- Out of Scope（Post-MVP）：進階報表、批量操作優化、進階搜尋/篩選優化、AI 相關功能、行動裝置 App（倉儲端，如未另行定義）。

## 主要非功能性需求（摘要）
- 效能：頁面載入 < 2 秒；互動回應 < 500ms；API 95% < 300ms。
- 即時性：狀態更新可見 < 3 秒；顏色/狀態視覺回饋 < 1 秒。
- 安全：登入驗證整合既有後台帳號密碼；會話安全登出；權限/廠區資料隔離。
- 可靠性：網路中斷提示；恢復後自動重新連線；關鍵操作可暫存並於恢復後同步（細節未定義）。

## 需釐清/未定義
- 會話逾時規則（逾時時間、續期策略）。
- 離線暫存的範圍、資料一致性與衝突處理規則。
- 推播通知的對象範圍（除運務外是否含其他角色）與失敗重試/回報規則。
