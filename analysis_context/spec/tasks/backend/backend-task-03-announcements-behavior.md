# 任務摘要
實作即時公告之列表查詢、篩選、單筆/批量已讀與刪除、詳細內容，並與後台同步。

# 來源 Spec
- 00-overview.spec.md（即時公告）
- 03-backend.spec.md（即時公告）

# 對應頁面名稱
- announcement-list、announcement-detail

# 責任範圍
- 公告列表：支援篩選，回傳含已讀/未讀狀態、標題、日期等；需依登入者資料範圍（若公告有廠區維度）過濾。
- 單筆/批量標記已讀、單筆/批量刪除；批量範圍依前端傳入之篩選條件。
- 單筆公告詳細（日期、時間、標題、內容）。
- 前台操作即時反映至後台（即時同步範疇）。

# 驗收標準
- 列表與詳細 API 行為符合 Spec；批量操作結果正確且與後台一致。
- 即時同步下後台異動可於時限內反映至前台。

# 備註
- 公告是否依廠區過濾依現有後台資料模型與需求決定；若有廠區欄位則需依登入者廠區過濾。
