# 任務摘要
實作碼頭清單、未排車次選項/列表、車輛任務狀態轉移與車次詳細資料之後端邏輯與 API。

# 來源 Spec
- 00-overview.spec.md（出貨作業管理）
- 03-backend.spec.md（出貨作業：狀態與資料責任）

# 對應頁面名稱
- shipping

# 責任範圍
- 碼頭清單：依登入者所屬廠區回傳啟用中之碼頭（來源：碼頭管理 > 碼頭列表）。
- 未排車次選項：當天配送日期、任務起點廠區＝登入者廠區、該任務有送貨；符合者任務狀態為待出貨；車次定義來自固定路線之車次代號；回傳格式支援 [車牌]([車次])(任務單號)。
- 未排車次列表：支援出貨日期、班別篩選；排除碼頭處理中、配送中、配送異常、配送完成；僅登入者廠區且有送貨之任務。
- 狀態轉移：選擇車次加入→碼頭處理中；完成出車/取消→配送中並自列表移除；缺貨待料/通知未到→對應狀態、自碼頭移除但保留於未排車次列表。
- 車次詳細：依車號回傳配送任務與配送點/配送單之收送貨、載具等詳細資料。

# 驗收標準
- 碼頭與未排車次資料正確過濾；狀態轉移後前端列表與儀錶板資料一致。
- 通知未到時觸發通知給運務員及運務部主管（見通知任務）。

# 備註
- 車次代號與任務/配送單資料來源為既有後台【派車任務管理 > 配送任務】等，需整合現有資料模型。
