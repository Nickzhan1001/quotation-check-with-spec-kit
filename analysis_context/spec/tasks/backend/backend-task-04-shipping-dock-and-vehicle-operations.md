# Task Summary
提供出貨作業所需後端資料與狀態變更：碼頭列表/狀態、未排車次列表、可選車次候選、車次詳細資訊、車輛狀態轉移驗證。

# Source Spec
- analysis_context/spec/00-overview.spec.md（出貨作業管理、即時同步）
- analysis_context/spec/03-backend.spec.md（出貨作業資料提供、狀態轉移規則）

# Responsibility
- Backend：出貨作業的資料提供與狀態寫入規則。

# Acceptance Criteria
- 後端可提供碼頭列表與碼頭顯示狀態供前台儀錶板使用。
- 後端可提供未排車次列表（可依條件查詢），且資料範圍符合內部廠區隔離。
- 後端可提供可選車次下拉候選，且候選符合條件：當天配送、同廠區、含送貨配送單、未完成出車且未取消。
- 後端可提供車次詳細資訊（基本資訊 + 配送點/客戶配送單明細）。
- 後端支援車輛狀態更新：正常、缺貨待料、通知未到、完成出車、取消；並驗證狀態轉移合法性（終態不可再變更）。

# Notes
- 「缺貨待料/通知未到」回退到未排車次集合的行為需與前台顯示一致。
