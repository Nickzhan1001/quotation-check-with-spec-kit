# 任務摘要
實作已排車次通知（發送給該車次運務員）、通知未到時通知運務員及運務部主管，以及出貨/收貨/退貨等即時同步機制。

# 來源 Spec
- 00-overview.spec.md（即時同步與通知）
- 03-backend.spec.md（即時同步與通知、出貨作業通知未到）

# 對應頁面名稱
- 支援全部即時更新之頁面（shipping、receiving-list、delivery-points-summary、return-list 等）

# 責任範圍
- 已排車次通知：接收前端觸發，發送推播至該車次配送任務之運務員。
- 通知未到：狀態改為通知未到時，發送推播至該車次運務員及運務部主管。
- 即時同步：後台操作 < 3 秒反映至前端；前端操作即時同步至後台與其他裝置；維持即時雙向連線（如 WebSocket 或等效）。
- 多裝置同步、連線中斷偵測與恢復；離線暫存與恢復後同步策略未定義。

# 驗收標準
- 通知可送達運務 APP；即時狀態變更在時限內於各端反映。
- 連線中斷與恢復有對應機制。

# 備註
- 推播失敗重試次數/間隔/降級為未定義；離線衝突處理規則為未定義。
