# 任務摘要
實作收貨列表、下貨狀態、回廠下貨回報接收、下貨碼頭選項、配送點總表與收貨明細（入庫分貨/越庫作業）之讀寫與排序規則。

# 來源 Spec
- 00-overview.spec.md（收貨作業管理）
- 03-backend.spec.md（收貨作業：狀態與資料責任）

# 對應頁面名稱
- receiving-list、delivery-points-summary、receiving-detail-inbound、receiving-detail-cross-dock

# 責任範圍
- 收貨列表：僅含有收貨且符合登入者廠區之任務；支援篩選；下貨狀態接貨中/回廠下貨/下貨完成；排序先回廠下貨再接貨中、同狀態依回報時間新到舊。
- 接收運務 APP 回廠下貨回報：更新狀態、記錄回報時間與位置、通知迄點同廠區碼頭人員、更新後台倉儲收貨資訊與前台列表。
- 下貨碼頭選項：回傳時排除該任務已選之出貨碼頭與下貨碼頭。
- 配送點總表：回傳該任務有收貨之配送單與收貨類型（入庫分貨/越庫作業）；下貨完成之任務不可再進入總表（校驗）。
- 收貨明細：入庫分貨/越庫作業之商品與可編輯欄位讀寫；多組至少保留一組之驗證；進入時間可供查詢更新。

# 驗收標準
- 列表、總表、明細資料與狀態正確；運務回報後狀態與通知正確；下貨碼頭選項排除正確。

# 備註
- 回報位置（經緯度轉地址）依 PRD 串接 Google Map API 或等效服務；經緯度來源為衛星犬 API。
