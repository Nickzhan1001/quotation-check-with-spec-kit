# 任務摘要
實作簽名送出後收貨狀態更新為下貨完成、與後台倉儲收貨資訊連動，以及簽名資料儲存與追溯。

# 來源 Spec
- 00-overview.spec.md（收貨作業管理）
- 03-backend.spec.md（簽名與收貨完成、審計）

# 對應頁面名稱
- delivery-points-summary

# 責任範圍
- 接收前端送出之簽名資料，驗證後將該任務收貨狀態更新為「下貨完成」。
- 與後台【倉儲收貨管理 > 倉儲收貨資訊查詢】之處理狀態連動。
- 簽名資料加密儲存並可追溯（審計）。

# 驗收標準
- 簽名送出後任務收貨狀態為下貨完成；後台查詢列表之處理狀態同步更新。
- 簽名紀錄可於審計需求下查詢。

# 備註
- 空白簽名是否拒絕為未定義，可與前端協調由前端阻擋或後端驗證拒絕。
