# 任務摘要
實作狀態變更與操作記錄、可追溯之審計日誌及與現有系統之資料一致性驗證與不一致回報。

# 來源 Spec
- 00-overview.spec.md（資料整合與審計）
- 03-backend.spec.md（資料整合與審計）

# 對應頁面名稱
- 全部頁面（審計紀錄與資料一致性）

# 責任範圍
- 所有狀態變更操作需記錄（含誰、何時、何項變更）；可供審計查詢。
- 與現有後台系統之資料一致性驗證；偵測並回報不一致（具體頻率與方式未定義）。
- 審計日誌保留期限未定義，實作時需與產品/法規協調並文件化。

# 驗收標準
- 關鍵狀態變更（出貨、收貨、退貨、簽名等）皆有紀錄可查。
- 具備與現有系統資料比對或偵測不一致之機制並可回報。

# 備註
- Blocked/未定義：審計保留期限、一致性檢查頻率與具體回報管道需釐清後實作或標註為未定義。
