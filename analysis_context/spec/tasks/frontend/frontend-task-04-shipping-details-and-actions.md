# 任務摘要
實作車次詳細配送單燈箱與狀態修改操作（完成出車、取消、缺貨待料、通知未到）及已排車次通知。

# 來源 Spec
- 00-overview.spec.md（出貨作業管理）
- 01-routing-pages.spec.md（出貨作業頁面）
- 02-frontend.spec.md（車次詳細配送單燈箱、狀態修改、通知圖示）

# 對應頁面名稱
- shipping

# 責任範圍
- 點擊碼頭已排車次開啟車次詳細燈箱；燈箱顯示基本資訊（上班時間、司機姓名、車號車次、路線別）與配送資訊（配送點清單可收合展開，配送點下配送單可收合展開）。
- 燈箱遮罩不覆蓋整個頁面；狀態修改按鈕（完成出車、缺貨待料、通知未到、取消）位於燈箱最下方。
- 完成出車：顯示「已完成出車」後關閉燈箱；其他按鈕關閉燈箱；送出中顯示載入並禁用其他操作。
- 已排車次之喇叭 icon：點擊發送通知，顯示「已通知司機前往碼頭」。

# 驗收標準
- 燈箱內容與收合展開行為符合 Spec；操作後狀態與列表/儀錶板即時更新（依後端同步）。
- 通知發送與狀態修改由後端處理，前端依回傳結果更新 UI。

# 備註
- 除「完成出車」外之按鈕操作後是否顯示額外提示文案為未定義，可依產品決定。
