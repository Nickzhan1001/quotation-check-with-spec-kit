# Task Summary
在出貨作業頁完成碼頭互動：未排車次下拉選單、已選定車次顯示車牌/喇叭 icon、車次詳細配送單燈箱（含可收合資訊與狀態操作按鈕）。

# Responsibility
- Frontend：碼頭互動、燈箱顯示與狀態操作後的畫面更新。

# Acceptance Criteria
- 未排車次的碼頭顯示下拉選單；選取後碼頭顯示車牌/車次與喇叭 icon。
- 點擊喇叭 icon 可觸發通知行為並顯示提示（提示文字內容未定義）。
- 點擊碼頭可開啟車次詳細配送單燈箱：
  - 遮罩不覆蓋整個頁面
  - 按鈕位於燈箱內最下方
  - 顯示基本資訊（上班時間/司機姓名/車號車次/路線別）
  - 配送資訊可收合/展開
  - 提供操作按鈕：取消、完成出車、缺貨待料、通知未到
- 狀態操作後的前台行為符合規格：
  - 完成出車/取消：車輛自碼頭儀錶板移除
  - 缺貨待料/通知未到：車輛回到未排車次列表，並可再次被選取；顏色同步更新

# Notes
- 通知未到需通知「運務部後台主管」的視覺回饋與文案未定義。
