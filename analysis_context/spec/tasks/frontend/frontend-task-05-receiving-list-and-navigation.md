# Task Summary
建立收貨列表頁：篩選、排序規則、下貨狀態顯示、回廠下貨時顯示回報位置/時間、下貨碼頭選擇、進入配送點總表的導航。

# Source Spec
- analysis_context/spec/01-routing-pages.spec.md（收貨列表頁、配送點總表入口）
- analysis_context/spec/02-frontend.spec.md（收貨列表行為、排序規則、下貨碼頭選擇規則、完成後不可進入）

# Responsibility
- Frontend：收貨列表頁面呈現與互動。

# Acceptance Criteria
- 收貨列表可依篩選條件更新顯示。
- 列表排序符合規格：先回廠下貨再接貨中；同狀態依回報時間新到舊。
- 回廠下貨狀態時，列表可顯示車輛回報位置與時間。
- 回廠下貨狀態時可選擇下貨碼頭（候選由後端提供，前台需能顯示與送出選擇）。
- 任務未完成時車號可點擊進入配送點總表；已完成時不可進入。

# Notes
- 下貨碼頭選項「自動排除已選定出貨和下貨碼頭」由後端提供/保證；前台僅需正確呈現。
