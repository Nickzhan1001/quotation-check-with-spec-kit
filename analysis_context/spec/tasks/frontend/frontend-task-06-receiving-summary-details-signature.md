# Task Summary
建立收貨作業的配送點總表與收貨明細頁，並完成簽名送出後的完成鎖定行為。

# Source Spec
- analysis_context/spec/01-routing-pages.spec.md（配送點總表頁、收貨明細頁、簽名板燈箱）
- analysis_context/spec/02-frontend.spec.md（入庫分貨/越庫作業行為、動態時間顯示、多組資料規則、簽名完成後鎖定）

# Responsibility
- Frontend：收貨總表/明細/簽名流程之畫面與互動。

# Acceptance Criteria
- 配送點總表可顯示配送點/客戶資訊與收貨類型（入庫分貨/越庫作業），並可點擊配送點進入收貨明細。
- 收貨明細進入後顯示進入時間並持續動態更新。
- 入庫分貨明細：可修改實收數量；可新增/刪除多組（最少保留一組）；每商品可填寫備註。
- 越庫作業明細：可填寫實收板位與備註；可新增/刪除多組（最少保留一組）。
- 配送點總表可觸發簽名板燈箱（簽名區至少 300x150px），支援清除與送出。
- 簽名送出成功後：返回收貨列表，任務狀態更新為已完成；已完成任務不可再次進入配送點總表。

# Notes
- 簽名送出失敗/重試的 UI 行為未定義。
