# 任務摘要
實作配送點總表、收貨明細（入庫分貨/越庫作業）與簽名板收貨完成流程。

# 來源 Spec
- 00-overview.spec.md（收貨作業管理）
- 01-routing-pages.spec.md（配送點總表、收貨明細入庫分貨/越庫作業）
- 02-frontend.spec.md（配送點總表、簽名板、收貨明細）

# 對應頁面名稱
- delivery-points-summary、receiving-detail-inbound、receiving-detail-cross-dock

# 責任範圍
- 配送點總表：顯示該任務所有有收貨之配送單（客戶、配送點、收貨類型）；點擊配送點進入收貨明細；收貨完成按鈕開啟簽名板。
- 簽名板：可繪製、清除、送出；送出中顯示載入並禁用；送出成功後關閉簽名板、狀態更新為下貨完成、返回收貨列表。
- 收貨明細：進入時間動態更新；入庫分貨可編輯實收數量、有效日期、備註，可新增多組（最少一組）；越庫作業可編輯實收板位、配送點、配送日期、備註，可新增多組（最少一組）。

# 驗收標準
- 總表與明細資料與導覽符合 Spec；簽名送出後任務收貨狀態變為下貨完成並返回列表。
- 下貨完成後該任務不可再進入配送點總表（列表上車號不可點擊）。

# 備註
- 簽名送出前是否允許空白簽名為未定義，可與後端協調由前端阻擋或後端拒絕。
